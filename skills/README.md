# Skills

This repository ships three primary skill surfaces.

## 1. `karpathy-llm-wiki-obsidian`
Use when defining or documenting the methodology, setting boundaries, or preparing the wiki workflow.

## 2. `obsidian-inbox-to-wiki-ingest`
Use when compiling one or more raw notes from `Inbox/` into `_wiki/`.

## 3. `obsidian-wiki-lint-triage`
Use when reviewing `_wiki/` health, surfacing repair priorities, and identifying backlog candidates.

## Installation guidance

How you install these depends on your host.

- If your host supports native skill files, copy each `SKILL.md` into the appropriate skill directory.
- If your host uses prompt snippets or recipes, adapt each skill into that format.
- Keep skill descriptions focused on **when to use the skill**, not on full workflow detail.

## Relationship between the three

- methodology skill first
- ingest skill for execution
- lint skill for maintenance

Do not collapse them into one mega-skill unless your host has no better reusable surface.
